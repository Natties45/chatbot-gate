# Version 2.3.0 — Auto KB Generation + Git Push

> **Status:** Planned
> **Date:** 2026-07-02
> **Scope:** Auto-generate structured KB entries (YAML) from closed cases every midnight.
  Push the generated entries to the GitHub knowledge repo under an `auto-generated/`
  folder. Add Settings tab for KB Auto-Generate with toggle, schedule, and manual
  trigger. Add git push capability to the existing git-sync service.

---

## Goals

- Automate knowledge capture: every midnight, the AI reads today's closed cases and
  generates structured YAML summaries for each, plus a daily digest.
- Write generated files to the local git repo at `/root/openstack-support/auto-generated/`.
- Push only the `auto-generated/` folder to GitHub (never push code changes).
- Add Scheduler system to run daily tasks (KB generation, file cleanup).
- Add Settings tab: KB Auto-Generate with enable/disable toggle, schedule time picker,
  manual "Generate & Push Now" button, status display, and logs.
- Keep machine-generated KB files separate from manually curated knowledge files
  (`auto-generated/` vs `knowledge/`).

---

## Core Decisions

| Topic | Decision |
|-------|----------|
| KB generation trigger | Scheduled: every midnight (configurable time in Settings). Manual trigger also available. |
| Output format | YAML (structured, machine-readable, consistent with existing KB format) |
| Output location | `/root/openstack-support/auto-generated/YYYY-MM-DD/` |
| Folder structure per day | `auto-generated/2026-07-02/daily-digest.yaml`, `cases/{caseId}-{role}.yaml`, `stats.yaml` |
| Git push scope | Only `auto-generated/` folder — never push `apps/`, `docker-compose.yml`, etc. |
| Git push branch | `main` |
| Git auth | Existing SSH key on server (`/root/.ssh/id_rsa` or `id_ed25519`) |
| Scheduler engine | `setInterval` in Next.js server (checks every 60s, runs at scheduled time) |
| Dedup prevention | DB lock flag (`Setting: kb-auto.lastRunAt`) prevents duplicate runs |
| LLM for summaries | Uses existing Groq primary / Ollama fallback router |
| Review workflow | Human reviews `auto-generated/` locally → manually curates into `knowledge/` → pushes back |

## Architecture

```
Every day at scheduled time (default 23:59):

  1. Scheduler fires (setInterval check)
     │
  2. Acquire DB lock (SET kb-auto.running = 'true')
     │
  3. Query: SELECT * FROM Case WHERE closedAt >= today 00:00 AND status = 'closed'
     │
  4. For each closed case:
     │  ├── Build prompt with case summary + full chat history
     │  ├── Call LLM: "Summarize this case as structured YAML"
     │  └── Parse YAML output, validate fields
     │
  5. Write files:
     │  ├── auto-generated/YYYY-MM-DD/daily-digest.yaml  (all cases summary)
     │  ├── auto-generated/YYYY-MM-DD/cases/DDMMYYNN-{role}.yaml  (per case)
     │  └── auto-generated/YYYY-MM-DD/stats.yaml  (daily statistics)
     │
  6. Git: cd /root/openstack-support
     │  ├── git add auto-generated/
     │  ├── git commit -m "auto: daily KB update YYYY-MM-DD"
     │  └── git push origin main
     │
  7. Release lock + log results
```

### Review Workflow (Human)

```
Next morning:
  1. git pull  → see auto-generated/YYYY-MM-DD/
  2. Read daily-digest.yaml  (overview of yesterday's cases)
  3. Review individual case YAMLs in cases/
  4. Curate: copy good entries into knowledge/vm-instance.yaml, etc.
  5. git add knowledge/; git commit -m "curate KB from YYYY-MM-DD"; git push
```

### 3. LLM Prompt for Summarization

The prompt for generating case YAML:

```markdown
You are a knowledge base curator for OpenLandscape Cloud (OLS).
Summarize the following support case into a structured YAML format.

## Case Details
Case ID: {{CASE_ID}}
Role: {{ROLE}}
Date: {{DATE}}
Chat Transcript:
{{CHAT_TRANSCRIPT}}
Case Summary: {{SUMMARY}}

## Output Format (YAML only, no markdown fences)
```yaml
case_id: "{{CASE_ID}}"
date: "{{DATE}}"
role: "{{ROLE}}"
category: "(one of 9 OLS categories)"
confidence: (0-100)
status: "(Resolved / Escalated / Incomplete)"
summary: "(1-2 sentence summary in Thai)"
resolution: |
  (step-by-step resolution actions)
root_cause: "(root cause if identified)"
kb_suggestion: "(what should be added to the KB from this case)"
tags:
  - tag1
  - tag2
```

Rules:
- category must be one of: VM/Instance, Network/Security, Account/Gate, Billing,
  Domain, SSL, File Storage, Abuse, Generic
- All text in Thai except technical terms
- Be concise but complete
```

### 4. Scheduler (`src/lib/scheduler.ts`)

```typescript
class Scheduler {
  private interval: NodeJS.Timeout | null = null;

  start() {
    this.interval = setInterval(() => this.tick(), 60_000); // every 60s
  }

  async tick() {
    const settings = await getSettings();
    const enabled = settings['kb-auto.enabled'] === 'true';
    if (!enabled) return;

    const scheduleTime = settings['kb-auto.scheduleTime'] || '23:59';
    const now = new Date();
    const currentTime = `${now.getHours().toString().padStart(2,'0')}:${now.getMinutes().toString().padStart(2,'0')}`;

    if (currentTime !== scheduleTime) return;

    // Prevent duplicate run
    const lastRun = settings['kb-auto.lastRunAt'];
    const today = now.toISOString().split('T')[0];
    if (lastRun && lastRun.startsWith(today)) return;

    try {
      await saveSetting('kb-auto.running', 'true');
      const result = await autoGenerateDailyKB();
      await saveSetting('kb-auto.lastRunAt', now.toISOString());
      await saveSetting('kb-auto.lastResult', JSON.stringify(result));
    } catch (error) {
      await saveSetting('kb-auto.lastError', String(error));
    } finally {
      await saveSetting('kb-auto.running', 'false');
    }
  }

  stop() { if (this.interval) clearInterval(this.interval); }
}
```

### 5. Git Push Capability (`src/lib/git-sync-service.ts` — edit)

**New action added: `push_auto_generated`**

```typescript
case 'push_auto_generated': {
  const date = new Date().toISOString().split('T')[0];
  const filesPattern = `auto-generated/${date}/`;

  // git add only the auto-generated folder
  const addRes = await runGitCmd(`git add ${filesPattern}`, localPath);
  if (addRes.error) throw new Error(`git add failed: ${addRes.stderr}`);

  // git commit
  const commitRes = await runGitCmd(
    `git commit -m "auto: daily KB update ${date}"`,
    localPath
  );
  // commit may fail if no changes — that's OK

  // git push
  const pushRes = await runGitCmd(`git push origin ${branch}`, localPath);
  if (pushRes.error) throw new Error(`git push failed: ${pushRes.stderr}`);

  break;
}
```

## Out Of Scope

- Pushing code changes (apps/, docker-compose.yml, etc.) — only auto-generated/ folder.
- Auto-curating AI output into knowledge/ (human review required).
- Semantic deduplication of KB entries (just raw generation).
- Real-time per-case generation (batch daily only).
- Multi-branch support (push to main only).
- KB index rebuilding (that's a separate knowledge-sync feature, already exists).
